# 个人贡献说明

## 个人定位

我在项目中主要负责策划工作，参与核心玩法、怪物与卡牌机制、地图和战斗体验设计；同时负责部分 Godot/C# 程序实现、Bug 修复、音频接入和测试报告整理。

## 主要贡献

- 核心玩法：参与设计“颜色链、换手、倍率”的核心战斗机制，推动机制从概念落到可玩规则。
- 卡牌与构筑：参与四色卡牌体系、费用节奏、卡牌功能定位和构筑体验设计。
- 怪物机制：参与一层怪物机制设计，完成部分怪物行为的程序接入与调试。
- 地图体验：参与六边形回廊地图、房间类型、路线推进和不可回头体验规划。
- 程序实现：在团队开发压力较大时完成部分 Godot/C# 功能实现和 Bug 修复。
- 音频接入：参与音频资源整理、接入和关键反馈节点调整。
- 测试整理：参与黑盒测试反馈整理、测试命令梳理、更新报告和合流说明撰写。

## 代表性问题处理

- 地图：处理景观地块、可达性、已访问房间不可重入和地图生成规则收口。
- 战斗：处理牌区同步、敌人回合稳定、倍率与状态上限等稳定性问题。
- 沙盒：处理战斗沙盒配置、显示溢出、自定义牌组和本地存档误判问题。
- 怪物：完成多种怪物机制的程序化整理，并接入现有敌人意图与行动架构。

## 能力体现

这个项目体现了我在系统策划、机制拆解、跨职能沟通、快速学习 Godot/C#、调试复杂玩法问题和整理团队交付材料方面的能力。
